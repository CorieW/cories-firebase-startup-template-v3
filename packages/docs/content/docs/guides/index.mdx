## Use guides for end-to-end workflows

Guides are the right place for multi-step flows that connect different parts of
the stack together, such as:

- first production deployment
- customizing brand and visual tokens
- documenting auth, billing, and admin flows

## Keep them practical

Prefer guides that help a reader finish a task, not just understand a concept.
Reference pages can hold exhaustive details; guides should build momentum.
