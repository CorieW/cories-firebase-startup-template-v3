## Build with the template, not around it

These docs are meant to be easy to grow with the project. Add a new `.mdx`
page, update the nearby `meta.json`, and it immediately joins the navigation,
table of contents, and search index.

<Steps>
  <Step>Write pages in MDX with a small frontmatter contract.</Step>
  <Step>

## Why this setup works

<Tabs items={['Theme', 'Authoring', 'Extensibility']}>
  <Tab value='Theme'>
    The docs app imports the same shared palette and document helpers as the
    dashboard and marketing packages, so typography, brand marks, and light and
    dark mode stay in the same family.
  </Tab>
  <Tab value='Authoring'>
    Content lives under `packages/docs/content/docs`, with `index.mdx` for
    landing pages and `meta.json` files to control section structure.
  </Tab>
  <Tab value='Extensibility'>
    Rich content is centralized through the MDX component registry, which keeps
    docs authoring simple while still allowing custom React-powered sections.
  </Tab>
</Tabs>
